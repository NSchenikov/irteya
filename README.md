# Irteya (React + TypeScript App)

Это приложение создано с использованием React и TypeScript в рамках тестового задания для трудоустройства.

Проект состоит из трех частей:
- **Задание 1:** Реализация динамической формы на основе JSON Schema (draft-7).
- **Задание 2:** Разработка DSL для хранения информации о графиках дашборда.
- **Задание 3:** Реализация эффективной функции подсчёта вхождений элементов массива.

---

## Установка и запуск

Клонируйте репозиторий:

```bash
git clone https://github.com/NSchenikov/irteya

Перейдите в папку проекта:
cd irteya

Установите зависимости:
npm install

Запустите приложение в режиме разработки:
npm run start

После запуска приложение будет доступно по адресу:
http://localhost:3000 (порт может отличаться в зависимости от настроек).

Доступные команды:
npm start – запуск dev-сервера.
npm run build – сборка production-версии в папку build.

Технологии:
React (функциональные компоненты, хуки)
TypeScript – статическая типизация

JSON Schema Draft-7 – описание структуры динамических форм
Сборка: Create React App (CRA)
Стили: CSS

Задание 1. Динамическая форма по JSON Schema
Реализована динамическая форма, основанная на структуре JSON Schema (draft-7).

Поддерживаемые типы данных:

string
integer
number
boolean
enum
object
array
Поддерживаемые правила валидации:

Обязательность полей (required)
Минимальная и максимальная длина строк (minLength, maxLength)
Минимальные и максимальные значения чисел (minimum, maximum)
Минимальное и максимальное количество элементов в массивах (minItems, maxItems)
Особенности:

Полная поддержка вложенности объектов и массивов.
Построение формы и валидация полностью управляются схемой.
Файлы решения:

Логика разбора схемы: src/utils/schemaParser.ts
Компоненты формы: src/components/FormBuilder/
Типы схемы: src/types/schema.ts

Задание 2. DSL для хранения информации о дашбордах
Разработан DSL (Domain Specific Language) для описания графиков, отображаемых на дашборде.

Описание структуры графика:

title: Название графика
type: Тип графика (linear, area, bar)
colors: Массив цветов линий
metrics: Названия метрик для отображения
Описание потока данных:

time: Время обновления данных (строка ISO)
metrics: Массив объектов { metricName: metricValue }
Файлы решения для второго задания:

Описание типов DSL: src/types/dsl.ts
Пример данных DSL: example/dashboard.dsl.json

Задание 3. Подсчёт вхождений элементов массива
Реализована функция для эффективного подсчёта количества вхождений элементов одного массива (A) в другом массиве (B).

Особенности решения:

Время выполнения алгоритма: O(N + M) (N — длина массива A, M — длина массива B).
Минимальное потребление памяти — используется хэш-таблица.
Обоснование и бенчмаркинг приведены отдельно.

Файлы решения для третьего задания:
Функция: src/algorithms/countOccurrences.ts
Тесты: src/algorithms/countOccurrences.test.ts
